— How to use this guide

Pick the use case closest to what you're making. Each card tells you which sections to open first, which dimensions to set up, which treatments to reach for, and the traps to avoid. If your project doesn't fit any of these, use Custom Visual as the fallback — the underlying tokens are the same.

slideshow
Use case 01
Deck / Presentation
Read first
Brand Foundation → Voice → Typography → Color. Skip components — most of them don't apply to slides.
Canvas
16:9 aspect ratio · 1920×1080 standard · content capped to 1100px wide · generous 56–80px margins on all sides.
Type
Lead with Display (80px) or H1 (56px). One headline per slide. Use the red italic emphasis on the most important word. Body text at 14–16px.
Components
Proof Card for results, Info Card for layer descriptions, Highlight Card for callouts. Eyebrow above every section title.
Color
Red Orange anchor (500) for emphasis only. White canvas for most slides, black surface for manifesto/hero moments. Secondary colors only if data viz requires.
Avoid
Small UI components (buttons, inputs, pills) — slides aren't interactive. Multiple emphasis words per headline. Filling every inch — let whitespace carry the rhythm.
web
Use case 02
Website / Product
Read first
Shell → Spacing → Components (all of them) → Anti-Patterns. The full stack matters here.
Canvas
Responsive · 1280px max for marketing, 1000px max for product UI · breakpoints at 320 / 768 / 1024 / 1280px.
Type
H2 (40px) opens sections, H4 (24px) is the workhorse for card titles. Body text at 13.5–16px. Use H6 (16px) for labels and metadata.
Components
All components apply. Stat Cards for KPIs, Info / Feature Cards for content, Buttons for actions, Pills for filters and tabs, Inputs for forms.
Color
Red for primary actions and accents only. Use secondary colors (lime for growth, blue for data) to differentiate categories. White or warm canvas for surfaces.
Avoid
Two primary buttons side by side. Display/H1 type inside content cards. Decorative gradients. Using red for everything — it should signal something.
share
Use case 03
Social Post
Read first
Voice (Signature Phrases) → Typography (Headline Pattern) → Color. Components are mostly irrelevant — this is a typography exercise.
Canvas
Square 1080×1080 (Instagram feed) · vertical 1080×1350 (Instagram portrait) · story 1080×1920 · LinkedIn 1200×627.
Type
Display or H1 only — anything smaller gets lost in the feed. One headline, one emphasis word. No body copy if avoidable.
Components
Eyebrow above the headline. Proof Card if you need a number. Nothing else — every component competes with the message.
Color
Black background with red emphasis = highest impact in feed. White background with red emphasis = editorial. Pick one, don't mix per post.
Avoid
Small text (below 24px renders unreadable at thumbnail). Stock imagery. Logos in three corners. More than 8 words in a headline.
auto_awesome
Use case 04
Custom Visual
Read first
Color (full wheel) → Spacing → Iconography. The foundation tokens give consistency even when there's no component to apply.
Canvas
Whatever the medium requires — but pick a canvas first and design within it. Don't let the visual sprawl. Apply the spacing scale as a grid if there's no obvious structure.
Type
DM Sans only (Serif for editorial moments). Use the heading scale as a reference — pick one tier and stick to it across the visual.
Treatments
Apply ramp stops from the color wheel consistently — pick a primary ramp and a supporting ramp, use 50/100 for tints, 500 for the anchor, 900 for the deepest contrast. Material Icons Round for any iconographic elements.
Color
Pick from the wheel — don't introduce new colors. If you need more than 3 colors, you probably need to simplify the visual instead.
Avoid
Custom typefaces. Outline icons (only Round). Gradients. Drop shadows on text. Off-system colors. Bypassing the spacing scale with "just this once" values.
The same rule applies to every use case: if you find yourself reaching for something that isn't in this guide — a new color, a new font weight, a new spacing value — pause and look again. The constraint is the point. The system works because it's narrow.

01 Brand Foundation

The four brand layers locked in the 2026 strategy. These are not creative briefs to interpret — they are fixed truths the entire design system serves. Every visual decision in this document traces back to one of these.

How to use this section. Before making any design decision — color choice, layout, copy tone — ask: does this support the brand essence (authentic commerce intent) and persona (catalyst, not operator)? If it feels like marketing filler or generic UI, it probably violates one of these layers.

02 Voice & Language

Ecomobi's voice is built from the brand foundation — declarative, evidence-led, momentum-oriented. The language patterns from the 2026 deck are the working examples for every surface that follows.

Signature Phrases
PhraseWhere it lives
Authentic Commerce IntentWhat we create — the customer outcome
Social Commerce VelocityHow we deliver it — speed, scale, compounding
Content Commerce CatalystWho we are — the brand persona
We don't run campaigns. We build momentum.Closing/manifesto line
Voice Principles
Never
  • Hedge with "we believe", "we strive", "we hope"
  • Lead with "Ecomobi is a leading..." — show, don't claim
  • Use enterprise abstraction ("synergy", "ecosystem", "solutions", "leverage")
  • Treat creators as "users" or "members" — they're creators
  • Describe SEA market dynamics — embody them
  • Manufactured enthusiasm ("amazing", "incredible", "revolutionary")
Always
  • Lead with the named outcome ("357% GMV uplift in 3 months")
  • Declarative sentences. Short. Confident.
  • Name the client, the number, the timeframe
  • "Creators" — that's our partner; name them
  • Let the proof carry the weight
  • End strong. No qualifying tail.
Headline Pattern

Use this pattern for hero headings, section titles, and any headline that needs to land with conviction. The rule is simple: write the headline in black bold, then italicise the most important word — it automatically turns red. Use sparingly — one emphasis per headline maximum. Red italic emphasis on the operative word inside a bold black headline. This is the most recognizable Ecomobi text treatment and should appear on every hero, every section header, and every key call-out.

All margin and padding values are drawn from the spacing scale. No arbitrary values. The hierarchy moves from large (page frame) → medium (section rhythm) → small (component internals).

The 24px rule. When in doubt, use 24px (token: lg). It is the most-used spacing value in the Ecomobi deck — the default card padding, the default section gap multiplier, the default between-element spacing at medium density. Start at 24px, then scale up or down by one step if the layout demands it.

15 Anti-Patterns
Color
Never
  • Pure black #000 — use black #1A1A1A
  • Pure white as page background — use canvas #F0EFED
  • Pure gray #808080 — use the gray ramp
  • Introduce blue, green, amber, or purple — not in the system
  • Two different reds in one surface — pick one
  • Use red for body text — only for emphasis, tags, accents
  • Use red-pale and red-light interchangeably — they're distinct surfaces
Always
  • red #F43C34 for brand mark, emphasis, tags, primary CTA
  • red-dark #C42A23 for hover/pressed CTA states
  • red-pale #FFF0EF for highlight surfaces and metric cards
  • canvas #F0EFED as the page background
  • white for cards lifted on canvas
  • Reference the deck tokens — they're the canonical truth
Typography
Never
  • Use Inter, Roboto, DM Sans, system-ui — only DM Sans / Serif
  • Italic without red — italic in Ecomobi always means emphasis
  • Headlines below weight 600
  • Body copy at weight 300 — too thin at small sizes
  • font-weight 600 in UI components (buttons, badges, labels) — UI uses 700 for emphasis, 500 for medium, 400 for body
  • Skip the tag eyebrow on a section header
  • Decorative numerals at full color saturation — they're chrome, keep them gray-200
  • Use IBM Plex Serif in UI, navigation, or buttons — editorial contexts only
Always
  • DM Sans everywhere in UI — 300 through 700
  • IBM Plex Serif for pull quotes and editorial moments only
  • Italic = red emphasis (the deck's signature move)
  • Headings H3–H6 at weight 600 · Display, H1, H2 at weight 700
  • UI components (buttons, badges, labels, nav) at weight 700 for emphasis, 500 for medium, 400 for body
  • Body text at weight 400 minimum
  • Tags before headers — every time
  • Tight letter-spacing on large display (−0.015 to −0.02em)
Voice
Never
  • "Leading", "innovative", "best-in-class" — show, don't claim
  • "Solutions", "ecosystem", "synergy", "leverage"
  • Refer to creators as "users" — they're creators
  • Hedge: "we hope", "we strive", "we believe"
  • End on a qualifier — "...if conditions are right." Cut it.
  • Generic stock imagery — handshakes, gears, abstract globes
Always
  • Lead with the proof — named client, real number, timeframe
  • Declarative sentences. Short. Confident.
  • Catalyst, velocity, compounds, system — the foundation vocabulary
  • "Authentic Commerce Intent", "Social Commerce Velocity", "Content Commerce Catalyst" — the locked phrases
  • End with momentum, not caveats
Iconography & Decoration
Never
  • Lucide, Heroicons, Tabler, Font Awesome, Feather — no third-party icon libraries
  • Emojis anywhere in presentations, website, or social posts
  • Gradients — solid color only
  • Drop shadows on text
  • Rounded image corners above 8px — the system ceiling for UI radius
  • 3D effects, glassmorphism, neumorphism
Always
  • Material Icons Round — sole icon system across presentations, website, social
  • Round variant only — never Outlined, Filled, Sharp, or Two-tone
  • Subtle shadows on cards only (deck's --shadow token)
  • Single circle decorations behind hero areas (the deck pattern)
